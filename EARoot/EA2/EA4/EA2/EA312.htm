Basic Path Basic Path
Details:
This use case begins when a Client requests a display of their account details. General account information is displayed (i.e. ID, username, billing address, delivery address etc). Command buttons are displayed to allow the user to view Open Orders or History. The use case terminates when the user selects the Exit or Back button.<br/>
Steps:
1. Client selects the command "View Account Details"<br/>
2. System retrieves Client account details from database  (eg. ID, username, billing address, delivery address, email address, etc.<br/>
3. System displays details<br/>
4. Client clicks on Back or Exit button and Use Case terminates.<br/>
Exceptions:
4a. View History (at 4)
4b. View Open Orders (at 4)
View History Alternate
Details:
If the user selects the View History command, the View History use case is executed.<br/>
Steps:
1. Client selects the command "View History"<br/>
2. System displays Client's Transaction History [Invokes: <a href="EA313.htm">View History</a>]<br/>
View Open Orders Alternate
Details:
If the View Open Orders command is selected, the View Open Orders use case is executed.<br/>
Steps:
1. Client selects the command "View Open Orders"<br/>
2. System displays the Clients currently active transactions. [Invokes: <a href="EA314.htm">View Open Orders</a>]<br/>
