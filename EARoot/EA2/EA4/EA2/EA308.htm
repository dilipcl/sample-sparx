Scenario Type
Basic Path Basic Path
Details:
1. Use case begins when the Client selects the "Create Account" command.<br/>2. The Client enters name, address, e-mail address, password and password confirmation.<br/>3. Client presses Submit button.<br/>4. Submitted details are validated.<br/>5. Processing message is displayed to the Client.<br/>6. New account is created in the database.<br/>7. Success message is displayed to Client.<br/><br/>
Steps:
1. Client selects the command "Create Account".<br/>
2. System prompts for Client details.<br/>
3. Client enters name, address, e-mail address, password and password confirmation.<br/>
4. Client clicks on "Submit".<br/>
5. System displays "processing" message.<br/>
6. System performs validation of Client details.<br/>
7. System creates new Client account in the database.<br/>
8. System displays "Success" message.<br/>
Exceptions:
4a. User Cancels Account Creation (at End)
7a. Validation fails (at 3)
8a. Account Creation Fails (at End)
No Submit Alternate
Details:
The Client may press the Exit or Back button at step 3. The use case terminates with no action performed.<br/>
Validation fails Exception
Details:
At step 4, if validation fails then an error message is displayed and the Client is returned to step 2 with the invalid field or fields highlighted.<br/>
Steps:
1. System displays error message "invalid user information".<br/>
2. Client acknowledges error message.<br/>
3. System returns to step 3 with invalid fields highlighted.<br/>
Cannot Create New Account Alternate
Details:
In step 6, if the database create fails, the Client is informed and the use case terminates.<br/>
Account Creation Fails Exception
Details:
Steps:
1. System displays error message: "Creation of User Account failed".<br/>
2. Client acknowledges error message.<br/>
3. System terminates Use Case.<br/>
User Cancels Account Creation Alternate
Details:
Steps:
1. Client clicks on "Exit" or "Back" button.<br/>
2. System displays message: "Are you sure?"<br/>
3. Client confirms.<br/>
4. System terminates the Use Case with no further actions performed.<br/>
