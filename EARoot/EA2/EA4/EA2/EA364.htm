Scenario Type
Client - Close Account Basic Path
Details:
The use case starts when a Client selects the Close Account command from the Account Details Screen. The system will load the Client's account details and check whether the Client has any outstanding orders. If not, the Client is shown a Confirmation Dialog. If the Client replies yes, the account is marked as closed. Note that none of the details are removed from the system, for auditing purposes.<br/>
Steps:
1. The Client selects "Close Account" command from the Account Details section.<br/>
2. System loads Client account details and checks for outstanding transactions.<br/>
3. System displays a confirmation dialog.<br/>
4. Client clicks on OK.<br/>
5. System marks the Client account as "Closed".  Note that for auditing purposes, client details are NOT removed from system.<br/>
Exceptions:
1a. Administrator - Close Account (at End)
3a. Outstanding Transactions (at End)
4a. User replies "No" to confirmation (at End)
Administrator - Close Account Alternate
Details:
This scenario is the same as "Basic Path - Client" except that the user is Administrator and instead of closing the account for the current user, it could be for any user of the system.<br/>
Steps:
1. Administrator is viewing account detail for selected client, selects "Close Account" command from the Account Details section.<br/>
2. System checks Client account for outstanding transactions.<br/>
3. System displays a confirmation dialog.<br/>
4. Administrator clicks on OK.<br/>
5. System marks the Client account as "Closed".  Note that for auditing purposes, client details are NOT removed from system.<br/>
User replies "No" to confirmation Alternate
Details:
If the user replies "No" to the "Are You Sure?" dialog, a message is displayed confirming that nothing has been changed and then the use case terminates.<br/>
Steps:
1. Client responds to confirmation prompt by clicking on "No".<br/>
2. System displays message "Account was NOT closed".<br/>
3. Client acknowledges message and Use Case terminates.<br/>
Outstanding Transactions Alternate
Details:
If the account has any outstanding transactions, a message is displayed to the effect that the account cannot be closed, giving the reasons.<br/>
Steps:
1. Systems displays message: "Cannot close account due to outstanding transactions".<br/>
2. Client acknowledges message and Use Case terminates.<br/>
